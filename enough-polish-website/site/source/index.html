What is J2ME Polish?

J2ME Polish is suite of tools for creating "polished" J2ME applications. Each tool meets a definite need of J2ME developers:
Build-tools with an integrated device-database, a powerful GUI, a framework for building localized applications, a game-engine, a logging framework and a collection of utilities.

These screenshots show the optional GUI of J2ME Polish in action. The pictures show the same application with two different designs. Have a look at the screenshots-page for more examples and details.

Build Tools

The build-tool takes care for the common tasks of preprocessing, compilation, obfuscation, preverification, packaging and JAD-creation. Thanks to the integrated device database, device-optimzation is easier than ever.

  • Device Selection: Select the devices for which the application should be build. This can be done explicitly by listing the devices or implicitly by stating requirements for the devices (e.g. "use all devices which support the MIDP/2.0 standard or the Mobile Media API").
  • Preprocessing: Create faster, leaner and device optimized applications with the power of J2ME Polish' preprocessing capabilities and the integrated device database. Check and use device specifications within your code without using hardcoded values or risking the portability of your application.
    You can use any device-capabilities which are defined in the device database: 
    //#if polish.api.mmapi || polish.midp2
	import javax.microedition.javax.microedition.media.*;
//#endif
  • Compilation & Preverifying: J2ME Polish compiles and preverifies everything automatically. No need to set APIs or the classpath or to define the float support of CLDC/1.1-devices manually.
    Many APIs like the Mobile Media API, the Wireless Messaging API, Nokia's User Interface API as well as the MIDP/1.0 and MIDP/2.0 APIs are already included into J2ME Polish for your convenience.
  • Obfuscation: J2ME Polish has build-in support for the ProGuard as well as the RetroGuard obfuscator, which both shrink and scramble your application code. Other obfuscators can be integrated easily.
  • JAR and JAD creation: J2ME Polish packages the application and creates the appropriate JAD file automatically. Custom JAD-attributes can be set, of course.
  • Extensible: Integrate third party obfuscators and use you own preprocessing directives.

Device Database

J2ME Polish includes a device database with over 150 J2ME devices. This database delivers the data for the powerful preprocessing capabilities of J2ME Polish. The database can be easily extended and is updated regularily.

The J2ME Polish GUI

J2ME Polish also offers a powerful GUI which works on MIDP/1.0 as well as on MIDP/2.0 devices and which can be designed using simple CSS-textfiles, a technique which every webdesigner masters.

  • CSS Design: Use the webstandard CSS for designing your J2ME application. Simple text files control the outer appearance of your application. Designers can work independently of the programmers at last! The design and all the content can be easiliy adjusted for specific devices or device groups.
  • Standards Compliant: The J2ME Polish GUI is fully compliant to the MIDP/2.0 standard. This means that you can use MIDP/2.0 features like a POPUP ChoiceGroup on MIDP/1.0 devices as well.
    Also the programmer does not need to learn a new API, and needs to use only the standard MIDP for coding - J2ME Polish "weaves" the necessary APIs during the build-process into the application.
  • Device Optimization: Most optimizations are needed for the user interface. Create standard MIDP applications and let J2ME Polish do the optimization work for you - automatically.
  • Optional & Controllable: The GUI can be turned off completely if desired and can be controlled by simple switches in the "build.xml". By default the GUI is not used for very old devices which have not the necessary capabilities for supporting the J2ME Polish GUI.
  • Powerful: Use animated backgrounds, fullscreen canvases, designable menu-bars and titles and more.
  • Extensible: You can extend the GUI with standard CustomItem and add your own backgrounds and borders.

Localization Framework

With J2ME Polish's localization framework you can easily create localized applications. In contrast to any other localization framework, applications which are localized with J2ME Polish have absolutely no overhead in size or performance compared with non-localized applications. You can localize messages, resources and use advanced features like date-formatting.

Game Engine

With J2ME Polish's game-engine you can use the MIDP/2.0 game-API for MIDP/1.0 devices as well. With one source code you can now deliver for all J2ME devices!

Logging Framework

With the logging framework simple System.out.println()-messages can be shown on real devices. The framework allows a fine grained control like the popular log4j-framework and can be disabled completely.

Utilities

Use some often needed utilities which are not available in the J2ME standard, e.g. the ArrayList or the text utility.

Status

J2ME Polish is a stable & complete solution for Windows, Linux and Mac OS X. With over 100 downloads per day, J2ME Polish is rapidly becoming the de-facto standard for developing J2ME applications. Please download the latest version (J2ME Polish <%= PolishVersion %>).
