Developing for Android Devices

The Android platform is a Java - driven platform for mobile devices from Google. In J2ME Polish, you can develop normal MIDlet applications and simply convert these applications to the native Android application format.

Preparations

To create and deploy application for Android all thats needed is the android sdk with the version 1.0 or higher. Please visit the Android Download Site to download the SDK for your operation system and install it.

Optionally, you can install the Android ADT Plugins for eclipse. For further information how to install the plugins, go to the Android Installation Guide.

After the installation of the Android SDK you should define the android.home in either ${polish.home}/global.properties, ${project.home}/${user.name}.properties or in your build.xml script. In your build.xml script the definitition could look like this: 

<property 
	name="android.home" 
	location="C:/Program Files/android-sdk-windows-1.0_r1" 
/>

Building Android Application

To convert, compile and package your source code for the Android platform, you simply need to set Generic/android as the device to build your application for. This device has a resolution of 240 x 320 and has all default features of an Android phone. Future versions of J2ME Polish will add more sofisticated devices like the T-Mobile G1.

The building of the application is done in steps :

  1. An activity is created in /activity of the Android build folder.
  2. The R.java / Manifest.java are created from the resources and sources via aapt of the Android SDK
  3. The code is compiled with target and source set to Java 1.5
  4. A file named classes.dex is created from the source code via dx of the Android SDK
  5. The resources are packed to a .ap_ file with aapt
  6. The application is packaged to a .apk file with the SDK's apkbuilder

Please note that the current android conversion is in its first stages. By now, only the Polish API and HTTP is supported. Other packages and feature will follow.

Testing Android application

There are two possibilities to test Android application build with J2ME Polish :

  • If test is set to true, the Android emulator is started automatically and the application is installed. Before launching the programm, please wait for the build.xml to output Successfully installed. To rebuild the application and test it, just cancel the running build.xml and leave the emulator running, the build process will automatically detect if there's an emulator running and install the application to the running emulator. You can also start the emulator before the build process is started.
  • The created activity in the build folder can also be imported to eclipse and used to test via the Android ADT Plugins if they're installed. Please note that you need to fix the project to use it with the ADT plugins. To do so, right-click the imported activity and run Android Tools / Fix Project Properties.

Troubleshooting

The creation of the .dex file terminates with an OutOfMemoryError

This error is quite common and due to insufficient memory settings in the dx executable of the android installation. Please go to ${android.home}/tools and open the dx executable. Remove REM from the line 

REM set javaOpts=-Xmx256M
and rerun the build process. If the error occurs again, increase the settings for -XmX to a higher amount.

Roadmap

Future versions of the android conversion will support the conversion of the following Java Packages :

  • RMS
  • Multimedia API
  • Location API
  • Bluetooth API
