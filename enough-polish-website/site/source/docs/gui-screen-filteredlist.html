The FilteredList allows you to limit the number of shown entries by entering some text. This speeds up the time for finding a specific entry in a long result list, for example.

Design

The FilteredList in the screenshot above utilized the fisheye view-type and has been realized using following CSS settings (without title and menubar settings):

.filterPopupStyle {
	repaint-previous-screen: true;
	margin: 10;
	margin-bottom: 0;
	padding: 5;
	background-color: black;
}

.filterTextFieldStyle {
	margin-left: 2;
	margin-right: 2;
	before: url( search.png );
	background {
		color: white;
		border-color: black;
	}
	layout: expand | left;
}

By default the TextField of a FilteredList is designed using the .filterTextFieldStyle.

CSS Attributes for Lists

You can use the following CSS attributes for a FilteredList:

<%cssattributes de.enough.polish.ui.FilteredList %>

CSS Attributes for items within a FilteredList

For styles of List items you can use the following CSS attributes:

<%cssattributes de.enough.polish.ui.ListItem %>

Programming

You can use the FilteredList like a normal javax.microedition.lcdui.List. Please refer to the List documentation for further details.

Configuration

You can configure if List should not show commands like "Select" for implicit and exclusive Lists or "Mark" for multiple lists. By default such commands are shown. Specify these configuration settings in the variables section of your build.xml script. Please refer to the List documentation for further details.
